Data Definition Language (DDL)

DDL CREATE TABLE Define a new relation and its schema
Formula
CREATE TABLE table_name ( col1 datatype [constraints], col2 datatype [constraints], ... PRIMARY KEY (col1), FOREIGN KEY (col2) REFERENCES other_table );
InputTable name, column names + types, optional constraints
OutputA new (empty) table added to the database schema
Key Data Types
  • char(n) — fixed length string
  • varchar(n) — variable length string
  • int / smallint — integers
  • numeric(p,d) — e.g., numeric(8,2) = 123456.78
  • float(n) / real / double — floating point
Simple Example
-- Create the instructor table CREATE TABLE instructor ( ID char(5), name varchar(20) NOT NULL, dept_name varchar(20), salary numeric(8,2), PRIMARY KEY (ID), FOREIGN KEY (dept_name) REFERENCES department );
When to use: At the start — before inserting any data. You're defining the skeleton of your data model.
DDL ALTER TABLE & DROP TABLE Modify or destroy an existing schema
Formulas
-- Add a new column (all rows get NULL) ALTER TABLE instructor ADD email varchar(50); -- Drop a column ALTER TABLE instructor DROP email; -- Delete the entire table (schema + data) DROP TABLE instructor;
When to use: ALTER when your schema needs to evolve after creation. DROP when you want to remove a table entirely. Note: DROP is irreversible — it removes the table AND all its data.
⚠ Warning: Dropping attributes is not supported in many databases.

Basic Query Structure: SELECT · FROM · WHERE

The three pillars of every SQL query.

SELECT A₁, A₂, ... -- What columns to return (projection) FROM R₁, R₂, ... -- Which tables (Cartesian product if multiple) WHERE condition; -- Filter rows (selection predicate)
QUERY SELECT clause — Projection Picks which columns appear in the result
Variants & Examples
-- All columns SELECT * FROM instructor; -- Specific columns SELECT name, salary FROM instructor; -- Arithmetic in select SELECT name, salary/12 AS monthly_salary FROM instructor; -- Remove duplicates with DISTINCT SELECT DISTINCT dept_name FROM instructor; -- Literal value (no table needed) SELECT 'Hello' AS greeting;
DISTINCT: SQL by default keeps duplicate rows. Use DISTINCT when you want unique values only — e.g., "what departments exist?" not "how many instructors per dept?"
AS (alias): Renames a column in the output. Use it to give computed columns a readable name, or to disambiguate columns from two tables.
QUERY FROM clause — Cartesian Product & Joins Lists source relations; multiple tables = cross product
-- Join instructor + course via matching dept_name SELECT instructor.name, course.title FROM instructor, course WHERE instructor.dept_name = course.dept_name;
Think of it like: FROM lists ALL the tables you need. Without a WHERE, you get every possible combination (Cartesian product — usually not what you want). Always add a WHERE join condition when using multiple tables.
QUERY WHERE clause — Filtering Rows Selection predicate: keeps only rows that satisfy the condition
-- Comparison operators: = < <= > >= <> SELECT name FROM instructor WHERE dept_name = 'CS' AND salary > 70000; -- BETWEEN (inclusive both ends) SELECT name FROM instructor WHERE salary BETWEEN 60000 AND 90000; -- Logical: AND, OR, NOT SELECT name FROM instructor WHERE dept_name = 'CS' OR dept_name = 'Biology';
When to use: Whenever you want a subset of rows. Think of WHERE as the "filter" step — it runs before SELECT chooses columns.

Additional Basic Operations

QUERY LIKE — Pattern Matching
-- % = any substring, _ = single char SELECT name FROM instructor WHERE name LIKE '%in%'; -- matches: Srinivasan, Einstein -- Exactly 3 characters WHERE name LIKE '___'; -- matches: Kim, Wu -- Escape literal % LIKE '100\%' ESCAPE '\';
Use when: You don't know the exact value but know part of it. Think "fuzzy search" for strings.
QUERY ORDER BY — Sorting Results
-- Sort by name A→Z (default ASC) SELECT name, salary FROM instructor ORDER BY name; -- Multiple sort keys ORDER BY dept_name ASC, salary DESC;
Use when: The output needs a specific order. ORDER BY runs last — after all filtering and projection. ASC = A→Z or low→high (default). DESC = reverse.
QUERY AS — Rename (Alias) / Self-Join Rename tables or columns; required when joining a table to itself
-- Column alias SELECT salary/12 AS monthly_sal FROM instructor; -- Table alias (Self-Join) -- Who earns more than ANY Biology instructor? SELECT DISTINCT T.name FROM instructor AS T, instructor AS S WHERE T.salary > S.salary AND S.dept_name = 'Biology';
When to use AS for self-join: Whenever you need to compare a table against itself — e.g., "find instructors who earn more than at least one Biology instructor." You must give the table two different aliases (T, S) so SQL can tell them apart.

Set Operations: UNION · INTERSECT · EXCEPT

Combine the results of two queries. Both queries must return the same number of columns with compatible types. Duplicates are automatically eliminated (use ALL variant to keep them).

SET UNION / INTERSECT / EXCEPT
Formulas
-- OR logic: rows in Q1 OR Q2 (query1) UNION (query2); -- AND logic: rows in Q1 AND Q2 (query1) INTERSECT (query2); -- MINUS: rows in Q1 but NOT Q2 (query1) EXCEPT (query2); -- Keep duplicates with ALL (query1) UNION ALL (query2);
Mental model:
🔵 UNION = "either or both"
🟢 INTERSECT = "both at the same time"
🔴 EXCEPT = "first but not second"

Requirement: Same number of columns, compatible types in each position.

NULL Values — The Three-Valued Logic

QUERY NULL — Unknown / Missing Value
-- Test for NULL — never use = NULL! SELECT student_id FROM takes WHERE grade IS NULL; -- ✓ correct -- WHERE grade = NULL ← ✗ WRONG: always unknown -- Arithmetic with NULL = NULL -- 5 + NULL = NULL -- NOT NULL check WHERE grade IS NOT NULL;
Three-Valued Logic Table
ABA AND BA OR B
trueunknownunknowntrue
falseunknownfalseunknown
unknownunknownunknownunknown
Key rule: A WHERE condition evaluates to unknown if any comparison involves NULL. Unknown is treated as false — so the row is excluded from the result.

Example: grade = 'A' where grade is NULL → unknown → row excluded.

Only IS NULL / IS NOT NULL can reliably test for nulls.

Aggregate Functions + GROUP BY + HAVING

AGG AVG · MIN · MAX · SUM · COUNT Collapse multiple rows into a single computed value
-- Count all rows SELECT COUNT(*) FROM instructor; -- → 12 -- Average salary (ignores NULLs) SELECT AVG(salary) FROM instructor WHERE dept_name = 'CS'; -- Max salary SELECT MAX(salary) FROM instructor; -- Count distinct values SELECT COUNT(DISTINCT dept_name) FROM instructor;
Use when you need a summary statistic about a group of rows, not individual row values.

⚠ NULL rule: All aggregate functions ignore NULL values — except COUNT(*) which counts all rows including NULLs.

📌 COUNT(*) = count rows. COUNT(col) = count non-null values in col. COUNT(DISTINCT col) = unique non-null values.
AGG GROUP BY — Aggregate Per Group
-- Average salary per department SELECT dept_name, AVG(salary) AS avg_sal FROM instructor GROUP BY dept_name; -- ❌ ILLEGAL — ID not in GROUP BY SELECT dept_name, ID, AVG(salary) FROM instructor GROUP BY dept_name; -- ERROR!
Golden rule: Every column in SELECT that is NOT inside an aggregate function MUST appear in the GROUP BY clause. Think: "what am I grouping by?" — those are your SELECT columns.
AGG HAVING — Filter on Groups
-- Departments with avg salary > 70,000 SELECT dept_name, AVG(salary) AS avg_sal FROM instructor GROUP BY dept_name HAVING AVG(salary) > 70000;
WHERE vs HAVING — the critical distinction:
🔵 WHERE filters individual rows before grouping.
🟣 HAVING filters groups after aggregation.

Think: "WHERE = row filter. HAVING = group filter."
You can't use aggregate functions in WHERE.

Nested Subqueries

A subquery is a SELECT query nested inside another query. It can appear in the WHERE, FROM, or SELECT clause.

SUBQ IN / NOT IN — Set Membership Test Tests whether a value belongs to a set (from a subquery or literal list)
-- Courses in Fall 2017 AND Spring 2018 SELECT DISTINCT course_id FROM section WHERE semester='Fall' AND year=2017 AND course_id IN (SELECT course_id FROM section WHERE semester='Spring' AND year=2018); -- Literal list: NOT IN SELECT name FROM instructor WHERE name NOT IN ('Mozart', 'Einstein');
When to use: "Is this value in some set?" — great for checking membership in another table's column.

IN ≡ = SOME (mathematically equivalent)
NOT IN ≡ ≠ ALL

⚠ Careful with NOT IN when the subquery can return NULLs — it will produce unexpected results. Use NOT EXISTS in that case.
SUBQ SOME (ANY) — At Least One Satisfies value op SOME(subquery) = true if condition holds for at least one row
-- Salary greater than at least one Biology instructor SELECT name FROM instructor WHERE salary > SOME (SELECT salary FROM instructor WHERE dept_name='Biology');
Logic
ExpressionMeaning
5 < SOME {0,5,6}true
5 < SOME {0,5}false
5 = SOME {0,5}true
= SOMEsame as IN
≠ SOMENOT same as NOT IN
When to use: "My value is [op] at least one value in this set." Useful for relative comparisons — "earn more than the cheapest department member."
SUBQ ALL — Every Row Satisfies value op ALL(subquery) = true if condition holds for every row
-- Salary greater than ALL Biology instructors SELECT name FROM instructor WHERE salary > ALL (SELECT salary FROM instructor WHERE dept_name='Biology');
Logic
ExpressionMeaning
5 < ALL {6,10}true
5 < ALL {4,6}false
≠ ALLsame as NOT IN
= ALLNOT same as IN
When to use: "My value is [op] every single value in this set." Classic use: "earn more than the highest-paid member of another dept" — stronger than SOME.
SUBQ EXISTS / NOT EXISTS — Emptiness Test Returns true if the subquery result is non-empty (or empty for NOT EXISTS)
-- Courses taught in both Fall 2017 AND Spring 2018 SELECT course_id FROM section AS S WHERE semester='Fall' AND year=2017 AND EXISTS ( SELECT * FROM section AS T WHERE semester='Spring' AND year=2018 AND S.course_id=T.course_id);
EXISTS: true if inner query returns ≥1 row.
NOT EXISTS: true if inner query returns 0 rows.

The inner query is a correlated subquery — it references the outer query (S.course_id). This runs for each row of the outer query.

NOT EXISTS trick: "Find students who took ALL Biology courses" = NOT EXISTS(Biology courses EXCEPT courses taken by student).
SUBQ UNIQUE — No Duplicate Test
-- Courses offered at most once in 2017 SELECT T.course_id FROM course AS T WHERE UNIQUE ( SELECT R.course_id FROM section AS R WHERE T.course_id=R.course_id AND R.year=2017);
When to use: UNIQUE returns true if the subquery has no duplicate rows. Useful for "offered at most once" type questions. Note: not supported in all databases (use COUNT instead as fallback).
SUBQ Subquery in FROM — Derived Table
-- Use a subquery as if it were a table SELECT dept_name, avg_sal FROM (SELECT dept_name, AVG(salary) AS avg_sal FROM instructor GROUP BY dept_name) AS dept_avg WHERE avg_sal > 70000;
When to use: When you need to filter or join on an aggregated result but can't use HAVING — or when you want to build a "temporary table" inline without the WITH clause. The subquery in FROM must be given an alias.
SUBQ WITH — Common Table Expression (CTE) Define a named temporary relation for use in the current query
-- Find depts whose total salary > avg dept total salary WITH dept_total (dept_name, total) AS (SELECT dept_name, SUM(salary) FROM instructor GROUP BY dept_name), dept_avg (avg_total) AS (SELECT AVG(total) FROM dept_total) SELECT dept_name FROM dept_total, dept_avg WHERE dept_total.total > dept_avg.avg_total;
When to use: When you need to reference the same subquery multiple times, or when a deeply nested query becomes hard to read. WITH gives each subquery a name and makes the query readable top-to-bottom like building blocks. Multiple CTEs are separated by commas.
SUBQ Scalar Subquery — Single Value Inline
-- For each dept, show count of instructors SELECT dept_name, (SELECT COUNT(*) FROM instructor WHERE instructor.dept_name = department.dept_name) AS num_instr FROM department;
When to use: In the SELECT clause when you need to add one extra computed column that requires looking up another table — without doing a full JOIN. The subquery MUST return exactly one row and one column. If it returns more → runtime error.

Modification: INSERT · DELETE · UPDATE

DML INSERT — Add New Rows
-- Insert single row (order matters) INSERT INTO course VALUES('CS-999','AI','CS',4); -- Insert with named columns (safer) INSERT INTO course (course_id,title, dept_name,credits) VALUES('CS-999','AI','CS',4); -- Insert from a SELECT query INSERT INTO instructor SELECT ID,name,dept_name,18000 FROM student WHERE dept_name='Music';
SELECT is fully evaluated before INSERT — so INSERT INTO t SELECT * FROM t is safe (no infinite loop).
DML DELETE — Remove Rows
-- Delete all rows (keep table) DELETE FROM instructor; -- Delete with condition DELETE FROM instructor WHERE dept_name='Finance'; -- Delete using subquery DELETE FROM instructor WHERE salary < (SELECT AVG(salary) FROM instructor);
Key: SQL first computes the subquery result (e.g., average), then deletes — it does NOT recompute as it deletes. DELETE removes rows; DROP TABLE removes the entire table.
DML UPDATE + CASE — Modify Values
-- Simple update UPDATE instructor SET salary = salary * 1.05 WHERE salary < 70000; -- Conditional update with CASE UPDATE instructor SET salary = CASE WHEN salary <= 100000 THEN salary*1.05 ELSE salary*1.03 END;
CASE in UPDATE: Avoids running two separate UPDATE statements where the second might affect rows just updated by the first. CASE handles all conditions in a single pass — much safer for conditional raises.
⚠ Order matters: Two separate UPDATEs for ≤100K and >100K could give a 100K instructor both raises if not careful. CASE solves this.

💡 Hint: Outer query scans section; EXISTS checks if matching Summer 2017 row exists

9

CHALLENGE — Find students who have taken ALL courses offered by the Biology department.

💡 Hint: NOT EXISTS ( Biology courses EXCEPT courses this student took ) — the "divide" pattern

-- Logic: a student has taken ALL Biology courses -- ⟺ there is NO Biology course they have NOT taken -- ⟺ NOT EXISTS (BioCourselist EXCEPT StudentCourseList) SELECT DISTINCT S.student_id FROM takes AS S WHERE NOT EXISTS ( (SELECT course_id FROM course WHERE dept_name = 'Biology') EXCEPT (SELECT T.course_id FROM takes AS T WHERE T.student_id = S.student_id)); -- Biology courses: BIO-101, BIO-301 -- S001 took BIO-101 only → excluded -- S002 took BIO-101 + BIO-301 → INCLUDED ✓
